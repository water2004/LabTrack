# LabTrack 项目方案

## 1. 系统概述
**LabTrack** 是一个轻量级实验室资产与效率管理工具。
*   **前端**：Vue 3 + Element Plus (PC/Mobile 自适应)。
*   **后端**：Python FastAPI + SQLite。
*   **核心逻辑**：用户免密登录、管理员单密码管控、实验设备自由组合、扫码即用。

---

## 2. 权限与认证设计 (Auth)

### 2.1 普通用户 (User)
*   **入口**：默认首页。
*   **认证**：**仅输入用户名**（如 "ZhangSan"）。
    *   后端校验是否存在该用户，存在则发放 Token，不存在则提示联系管理员。
*   **权限**：搜索设备、自由组合实验、扫码开始/结束、查看自己状态。

### 2.2 管理员 (Admin)
*   **入口**：点击首页底部的“管理后台”链接，或访问 `/admin`。
*   **认证**：**仅输入系统密码**（如 "LabAdmin2024"）。
    *   *无用户名*。系统设有一个全局管理密码（可在后端配置文件或环境变量中设置）。
*   **权限**：用户名单管理、设备增删改、强制终止实验、导出 Excel 报表。

## 3. 核心功能模块

### 3.1 设备库与搜索 (Device & Search)
*   **展示形式**：
    *   **卡片视图**（移动端/PC）：显示设备图、名称、状态点（🟢/🔴/🟡）、负责人。
    *   **列表视图**（管理后台）：显示详细表格。
*   **搜索功能**：
    *   顶部有一个大搜索框。
    *   **逻辑**：输入关键词，后端同时匹配 `device_name`（设备名） OR `asset_code`（固定资产标签）。
    *   *场景*：用户手里拿着贴着 "ZC-001" 标签的仪器，直接搜 "001" 即可找到。

### 3.2 实验定制与执行 (User Workflow)

这是本系统的核心交互，不再依赖后台模板，完全由用户自主操作。

**场景一：自由组合实验 (购物车模式)**
1.  用户登录。
2.  在设备列表中，点击设备卡片上的“**+ 加入实验**”按钮（类似加入购物车）。
3.  底部浮层显示：“已选设备 (3)：离心机、摇床、移液枪”。
4.  用户点击“**开始实验**”。
    *   **步骤 A (执行)**：系统锁定这 3 台设备，记录开始时间。
    *   **步骤 B (分享)**：系统生成一个当前组合的链接（如 `.../start?ids=1,5,9`），用户可收藏该链接，下次直接点击链接即可一键复用此组合。

**场景二：扫码直连 (单机模式)**
1.  用户扫描设备身上的二维码。
2.  跳转至该设备详情页。
3.  点击“**立即开始**”。

**场景三：状态终止**
1.  用户首页顶部显示“当前进行中的实验（2台设备）”。
2.  点击“**结束实验**”。
3.  系统释放资源，并写入 2 条使用记录。

### 3.3 后台管理 (Admin Dashboard)
*   **设备管理**：
    *   新增/编辑：上传图片（手机拍）、录入固定资产编号、指定负责人。
    *   **打印二维码**：为每台设备生成独立二维码，用于张贴。
*   **用户管理**：
    *   简单的名单维护（姓名）。只有在名单里的人才能在前台登录。
*   **数据导出**：
    *   选择时间范围 -> 下载 `.xlsx`。
    *   Excel 内容：资产编号 | 设备名 | 使用者 | 开始时间 | 结束时间 | 时长 | 备注。

## 4. 数据库设计 (SQLite)

### 4.1 Users (用户表)
| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | Integer | 主键 |
| `username` | String | 唯一，登录凭证 |
| `created_at` | DateTime | 注册时间 |

### 4.2 SystemConfig (系统配置表 - 可选)
*如果不想把管理员密码写在代码里，可以存在这里*
| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `key` | String | 例如 "admin_password_hash" |
| `value` | String | 密码的哈希值 |

### 4.3 Equipments (设备表)
| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | Integer | 主键 |
| `asset_code` | String | **固定资产编号** (唯一索引，用于搜索) |
| `name` | String | 设备名称 (用于搜索) |
| `image_path` | String | 图片路径 |
| `location` | String | 存放位置 |
| `manager` | String | 负责人 |
| `status` | Integer | 0:闲置, 1:使用中, 2:借出中 |

### 4.4 ActiveSessions (当前活跃记录)
| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | Integer | 主键 |
| `user_id` | Integer | 谁在用 |
| `equipment_id` | Integer | 哪个设备 |
| `start_time` | DateTime | 开始时间 |
| `group_id` | String | (可选) UUID，用于标识这几个设备是属于同一次“组合实验”开启的 |

### 4.5 UsageRecords (历史归档)
| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | Integer | 主键 |
| `asset_code` | String | 资产编号 |
| `device_name`| String | 设备名快照 |
| `user_name` | String | 使用者名快照 |
| `start_time` | DateTime | 开始 |
| `end_time` | DateTime | 结束 |
| `duration` | Integer | 秒数 |

## 5. 接口设计 (API)

### User APIs
*   `POST /auth/login`: Body: `{username: "xxx"}` -> Returns Token
*   `GET /devices`: Params: `q=search_term` (支持名称或资产号)
*   `POST /experiment/start`: Body: `{device_ids: [1, 2, 3]}`
    *   *逻辑*：检查所有 ID 是否闲置 -> 全部锁定 -> 返回成功。
*   `POST /experiment/stop`: Body: `{device_ids: [...]}`
*   `GET /users/me/active`: 获取我当前占用的设备列表。

### Admin APIs (Header需携带 AdminToken)
*   `POST /admin/auth`: Body: `{password: "123456"}` -> Returns AdminToken
*   `POST /admin/users`: 增加用户
*   `POST /admin/devices`: 增加设备
*   `GET /admin/export`: 导出 Excel 流

## 6. 前端交互细节 (Vue)

### 6.1 首页 (Home)
*   **顶部栏**：Logo + 搜索框 + 用户名(登出)。
*   **状态栏**：如果用户有正在进行的实验，顶部出现一个浮动横条：“正在使用 3 台设备 [点击结束]”。
*   **设备列表**：
    *   每个卡片右下角有一个 `Checkbox` 或者 `+` 号。
    *   选中后，屏幕右下角出现 `Floating Action Button (FAB)`，显示“已选 (2) -> 去开始”。

### 6.2 扫码页 (Scan)
*   解析 URL 参数获取 DeviceID。
*   **未登录**：提示输入用户名登录，登录后自动跳回此页。
*   **已登录**：
    *   设备闲置：显示大按钮“开始使用”。
    *   设备忙碌：显示“张三 (正在使用)”，本人可强制结束（或只能结束自己的，看管理规定）。

### 6.3 后台登录 (Admin Login)
*   界面极简，屏幕中央只有一个密码输入框 `type="password"` 和一个“验证”按钮。
